# agpo

> **AG**ent **P**lugin **O**rchestrator

Scaffold and manage **plugin marketplaces for Claude Code, GitHub Copilot, OpenAI Codex, and Cursor**, distributed through **any Git host** — GitHub, GitLab, Bitbucket, Gitea, or a self-hosted forge.

Think `ng` for Angular, but for agent plugin marketplaces: `init` gives you a push-ready repository, and `add`/`build`/`sync`/`bump`/`validate` cover the whole life of the project afterwards.

> Unofficial community tool, not affiliated with Anthropic, GitHub, OpenAI, or Cursor.

## How agents consume your marketplace

agpo always writes one canonical catalog — `.claude-plugin/marketplace.json` — plus your plugins under `plugins/<name>/`. That catalog is the single source of truth. How each agent reads it falls into two groups:

- **Native (no build step)** — **Claude Code** and **GitHub Copilot** read `.claude-plugin/marketplace.json` directly. Push the repo and it installs.
- **Generated registry (`agpo build`)** — **Codex** and **Cursor** install from their own committed registry, which `agpo build` derives from the same canonical catalog. You run `agpo build` once to enable a target; after that `add`/`bump`/`sync` keep it fresh automatically.

You can target several agents from a single repository — the plugin content under `plugins/` is shared.

## Quick start

```bash
npx agpo init my-marketplace
cd my-marketplace
agpo add skill tdd-coach
agpo validate
git remote add origin <your-git-url>
git add -A && git commit -m "feat: initial marketplace" && git push -u origin main
```

The default init targets Claude Code and GitHub Copilot (both native). To target Codex or Cursor, see below.

## Create a marketplace for your agent

Choose targets at init with `--agents`. Every flow ends with a normal git push; distribution works from any forge.

### Claude Code

Native — nothing to generate.

```bash
agpo init my-marketplace --agents claude-code
cd my-marketplace
agpo add skill my-skill
agpo validate
git remote add origin <your-git-url>
git add -A && git commit -m "feat: initial marketplace" && git push -u origin main
```

Consumers install in a Claude Code session:

```
/plugin marketplace add <owner/repo or git URL>
/plugin install my-skill@my-marketplace
```

### GitHub Copilot

Native — same repository as Claude Code, no build. The default init already includes Copilot; to target it explicitly use `--agents copilot` (or `--agents claude-code,copilot`).

```bash
agpo init my-marketplace --agents claude-code,copilot
cd my-marketplace
agpo add skill my-skill
agpo validate
git add -A && git commit -m "feat: initial marketplace" && git push -u origin main
```

Consumers install from the shell (or with `/plugin marketplace add` inside a session):

```bash
copilot plugin marketplace add <owner/repo or git URL>
copilot plugin install my-skill@my-marketplace
```

### OpenAI Codex

Codex installs from its own registry, which `agpo build` generates from the catalog.

```bash
agpo init my-marketplace --agents codex
cd my-marketplace
agpo add skill my-skill
agpo build          # generates .agents/plugins/marketplace.json + plugins/<name>/.codex-plugin/plugin.json
agpo validate
git add -A && git commit -m "feat: initial marketplace" && git push -u origin main
```

Consumers install with:

```bash
codex plugin marketplace add <owner/repo or git URL>
```

The Codex registry format follows the official Codex plugin docs (`.agents/plugins/marketplace.json` with `source: { source: "local", path }`, `policy`, and `category`). Commit the generated files so consumers can install.

### Cursor

Cursor installs from a committed `.cursor-plugin/` registry, also generated by `agpo build`.

```bash
agpo init my-marketplace --agents cursor
cd my-marketplace
agpo add skill my-skill
agpo build          # generates .cursor-plugin/marketplace.json + plugins/<name>/.cursor-plugin/plugin.json
agpo validate
git add -A && git commit -m "feat: initial marketplace" && git push -u origin main
```

Consumers add the marketplace in Cursor, then install the plugin by name.

### Several agents at once

One repository can serve every supported agent:

```bash
agpo init my-marketplace --agents claude-code,copilot,codex,cursor
cd my-marketplace
agpo add skill my-skill
agpo build          # generates the Codex and Cursor registries; Claude Code and Copilot need none
agpo validate
git add -A && git commit -m "feat: initial marketplace" && git push -u origin main
```

## Commands

| Command | What it does |
| :------ | :----------- |
| `agpo init [dir]` | Scaffolds a git-first marketplace: `.claude-plugin/marketplace.json` (with `$schema`, `pluginRoot`, `targets`), `plugins/`, `AGENTS.md`, a per-agent README, `examples/team-settings.json`, CI (GitHub Actions or GitLab CI), `.gitignore`, `git init`. Pick target agents with `--agents` (see [Target agents](#target-agents)). Non-interactive with `-y`. |
| `agpo add <template\|spec> <name>` | Scaffolds a plugin from a built-in template (`skill`, `command`, `agent`, `hook`, `mcp`) **or a remote/local one** — `gh:owner/repo/dir#ref`, `gl:owner/repo`, any git URL with `//subdir` and `#ref`, or a local path. Registers it in the catalog and refreshes the README plugin table. |
| `agpo build [--target] [--check]` | Generates the registries for **Codex** and **Cursor** from the catalog: their committed `marketplace.json` + per-plugin manifest mirrors. Default targets are the ones in `metadata.targets`; `--target codex,cursor` overrides. `--check` fails on drift (CI). |
| `agpo bump [plugin] [level]` | Bumps a plugin version from conventional commits scoped to its directory since the last `<plugin>@x.y.z` tag (`feat`→minor, breaking→major, else patch), or an explicit `major\|minor\|patch`. `--tag` commits and tags; `--dry-run` previews. Catalog, README, and any built registries stay in sync. |
| `agpo sync` | Reconciles the catalog with what's on disk. Source of truth: each plugin's `.claude-plugin/plugin.json`. Adds missing entries, fixes drift, flags orphans, regenerates the README table, and refreshes any already-built Codex/Cursor registry. |
| `agpo validate [--strict]` | Local checks (JSON validity, kebab-case, reserved names, source resolution, manifest presence, version drift) plus delegation to `claude plugin validate` when the Claude Code CLI is installed (`--strict` forwarded). Non-zero exit on error — CI-ready. |
| `agpo list` | Lists available plugin templates. |

## Target agents

`agpo init --agents <list>` records the agents you want to serve and adapts the scaffold. Default: `claude-code,copilot`.

| Agents | How agpo serves them |
| :----- | :------------------- |
| **Claude Code, GitHub Copilot** | Native. The catalog is read as-is — per-agent install docs and the team-settings path are wired at init, nothing to generate. |
| **Codex, Cursor** | `agpo build` generates the committed registry + per-plugin manifest mirrors from the catalog. `sync`/`add`/`bump` keep it fresh once built. |

Every init also writes a root `AGENTS.md`, read natively by most agents for repository context.

## Keeping registries fresh

Generation is opt-in: run `agpo build` once to enable Codex/Cursor. From then on, `agpo add`, `agpo bump`, and `agpo sync` refresh the generated registries automatically, so they never drift as plugins change. In CI, run `agpo build --check` to fail the build if a committed registry is out of date (the generated GitHub Actions / GitLab CI workflows already do this).

## Team templates (remote or local)

Version your team's plugin templates in any git repository and consume them from every marketplace:

```bash
agpo add gh:my-org/plugin-templates/kata-fr gilded-rose
agpo add "https://gitlab.company.io/craft/templates//skill-ddd#v2" tactical-ddd
agpo add ./shared-templates/hook-guard no-secrets
```

A template is any directory containing `.claude-plugin/plugin.json` (adopted, `name` rewritten) or `plugin.json.tpl`. Files ending in `.tpl` are rendered with `{{pluginName}}`, `{{pluginTitle}}`, `{{description}}`, `{{authorName}}` — in contents **and** in file/directory names. Executable bits are preserved.

## Forge-agnostic by design

`init` parses your git remote (https, ssh, scp-style) regardless of host. On github.com it uses the `owner/repo` shorthand and `{"source": "github"}` team settings; everywhere else it emits the git URL form — which every supported agent accepts for any forge.

## Team setup (automatic installation)

`init` writes `examples/team-settings.json` with an `extraKnownMarketplaces` block. Drop it into the settings file for each agent so teammates get the marketplace automatically when they trust the repository — `.claude/settings.json` for Claude Code, `.github/copilot/settings.json` for GitHub Copilot.
